    Integration


      The Integration module allows you to seamlessly connect the Expense Tracker with third-party applications to enhance functionality and automate workflows. One of the key integrations available is with Zoho Books.



      Quickbooks Integration


        Quickbooks is a widely used cloud-based accounting platform by Intuit that enables businesses to manage invoices, payroll, expense tracking, tax compliance, bank reconciliation, and financial reporting. Integrating it with the Expense Tracker application helps automate workflows and reduce manual entry.

        To configure, go to Account Integration and select Quickbooks.


        This will open the Quickbooks Integration Panel.


        The panel includes the following options:

        • Enable Quickbooks — Toggle to activate or deactivate the integration.
        • Add User in Quickbooks — Allow adding users directly into Quickbooks.
        • Quickbooks Client ID — Unique identifier for API authentication.
        • Quickbooks Client Secret — Secret key paired with Client ID for secure API access.
        • Quickbooks Sandbox Company ID (Company ID) — Unique identifier of your Quickbooks company account.
        • Quickbooks Request URL — URL used for authentication and authorization.
        • Redirect URI for Approved — Redirect link after successful authorization.
        • Redirect URI for Settings — Redirect link to manage integration settings.

        Once all the required details are filled in, click Save to apply and store the Quickbooks integration settings.


        How to Get Quickbooks Credentials

        Step 1: Client ID & Secret
        1. Go to the Intuit Developer Portal and sign in with your Quickbooks account.


        2. Create a new app by selecting Quickbooks Online and Payments.
        3. Fill in application details such as app name, add Permissions.


        5. After creating the app, copy the Client ID and Client Secret.


        Step 2: Sandbox Company ID (Company ID)

        2. In the top menu, go to MyHub, open the dropdown, and select Sandbox.


        4. Your Sandbox Company ID (Company ID) will be displayed in the URL or under your company profile.


        5. Copy and paste this Sandbox Company ID into the integration panel.


        Step 3: Request URL
        1. Open Power Automate and navigate to Solutions.


        2. Search for HR365 Expense Tracker and open it.

        3. Click on Cloud Flows, then select Quickbooks and click on the edit (pencil) icon.

        4. Select Manual and copy the HTTP URL.


        5. Paste this URL into the Quickbooks Request URL field in Expense Tracker.


        Step 4: Redirect URIs
        Enter the Expense Tracker site URL in both Redirect URI fields: - Redirect URI for Approved - Redirect URI for Settings Then click Save to complete setup.


        To manage users, click on Users in the side panel, then go to Settings.


        In the settings panel, scroll down to Validate Users with Quickbooks and click Configure.


        Next, click Sync Users from Quickbooks to import users.


        For mapping expenses, open the Expense Type panel and select Quickbooks Expense Type.


        In the Quickbooks Expense Type panel, map your application’s master expense types to the respective Quickbooks expense categories.


        Once an expense is submitted and approved by the final approver, a confirmation prompt will appear.


        After approval, the expense record is automatically marked as Expense Synced in Quickbooks.


        The synced details will then be visible directly in the Quickbooks Online account.


      Plaid Credit Card Expense Integration

      This document details the configuration and workflow for the Plaid Integration within the Expense 365 application, allowing users to import credit card transactions directly into their expense reports.

      Sign Up and Login Setup

      Step 1 — Create a Plaid Developer Account

      Go to https://dashboard.plaid.com/signup in your browser. You will see a sign-up form with the following fields:

      Field What to Enter
      First Name / Last Name Your full name
      Work Email Use a real email address — Plaid sends a verification link here
      Password At least 8 characters, mix of letters and numbers
      Company Name Your app or company name (e.g. 'Expense Tracker App')
      Company Size Choose the option that matches your team size
      Use Case Select 'Personal Finance / Budgeting' or the closest option

      Click Create Account. Plaid will send a verification email — open it and click the confirmation link to activate your account.





      Step 2 — Log In to the Plaid Dashboard

      Once your email is verified, go to https://dashboard.plaid.com and log in with your email and password. You will land on the Dashboard home page.


      Step 3 — Find Your API Keys (Client ID & Secret)

      Your Client ID and Secret keys are under the Team Settings section. Follow these steps exactly:

      1. Click your profile icon or company name in the top-left corner of the Dashboard
      2. Select Team Settings from the dropdown menu
      3. Click the Keys tab in the left sidebar
      4. You will see your credentials listed on this page

      Credential Where It Appears & What It Looks Like
      Client ID Shown at the top of the Keys page. It is always visible. Example: 5f3a2b1c4d8e9f0a1b2c3d4e
      Sandbox Secret Listed under the Sandbox section. Click the eye icon to reveal it. Copy it immediately.
      Development Secret Listed under the Development section. Only visible after you apply for Development access.
      Production Secret Listed under the Production section. Only visible after Plaid approves Production access.

      Step 7 — Set Allowed Redirect URIs (for OAuth banks)

      Some banks (like Chase or Wells Fargo) use OAuth, which redirects the user to the bank's website and then back to your app. For these flows, you must register your redirect URL in the Plaid Dashboard:

      1. Go to Team Settings → API in the Dashboard
      2. Under Allowed redirect URIs, click Add URI
      3. For local development, add: http://localhost:3000/oauth-callback
      4. For production, add your live domain: https://yourdomain.com/oauth-callback

      Environment Purpose & Details
      sandbox Free. Uses fake bank data. No real accounts, no approval needed. Use this while building and testing. Secret is available immediately after signup.
      development Free up to 100 live accounts. Connects to real banks. Requires you to submit a Development access request from the Dashboard. Plaid usually approves within 1-2 business days.
      production Paid (per API call). Required for public launch. Requires a separate Production access application and agreement to Plaid's terms of service.

      1. Configuration & Setup

      To enable automated credit card transaction fetching, administrators must configure the connection to the Plaid service.

      Account Integration:

      1. Navigate to the Settings (gear icon) and locate the Integration section.
      2. Select Account Integration to open the configuration panel.
      3. Toggle Plaid to On.
      4. Enter the following required credentials provided by Plaid:
        • Client ID
        • Secret ID
        • Request URL
        • Environment
        • Redirect URI
      5. Click Save to finalize the connection.

      2. Credit Card Expense Workflow

      Once integrated, users can pull live transaction data from their linked bank accounts into the expense system.

      Initiate Import:

      1. From the home dashboard, click the + Add dropdown menu.
      2. Select Credit Card Expense.

      Bank Connection (Plaid Link):

      • A Plaid authentication popup will appear.
      • Authentication: Users enter their phone number to receive a security code or log in directly to the Plaid service.
      • Account Selection: Once authenticated, users select the specific institution (e.g., Tartan Bank or First Gingham Credit Union) and the accounts they wish to share.
      • Click Confirm to authorize the data sync.

      Transaction Re-fetching:

      • After authorization, the system may prompt: "Additional transactions have been identified. Would you like to fetch more?"
      • Upon clicking Confirm, a loading modal appears with the status: "Please wait, your transactions are being re fetched..."



      3. Multiple Expense Processing

      Once transactions are successfully fetched, the user is redirected to the Multiple Expense view to categorize and finalize the data.

      • Data Redirection: Imported transactions (Date, Description, and Total Amount) are automatically populated as line items in the Multiple Expense grid.
      • Categorization: Users must manually select the Expense Type for each imported line item (e.g., Travel, Food, or Transportation).
      • Validation: Negative values (refunds/credits) and positive charges are both imported for review.
      • Submission: After verifying the details and adding any necessary Expense Attachments, users can submit the collective report.

      Zoho Books Integration


        Zoho Books is a cloud-based accounting platform that helps businesses manage invoicing, expense tracking, inventory, tax compliance, bank reconciliation, and financial reporting. It reduces manual errors and provides insights for better financial decision-making.

        To enable the integration, go to Integration then click on Account Integration, turn on the Zoho Books toggle this will open the Zoho Books integration panel.


        Step 1: Client ID & Secret
        Log in to https://api-console.zoho.in. Enter the Expense Tracker site URL in both Homepage and Authorized Redirect fields.


        Then navigate to Client secret, Copy the Client ID and Client Secret.


        Step 2: Organization ID
        Log in to https://books.zoho.in, click on the company name, and copy the Organization ID.


        Step 3: Request URL
        Configure Power Automate to obtain the Zoho Books Request URL. In Power Automate, open Solutions, search for HR365 Expense Tracker and open it.


        Then search Zoho and click on edit icon.


        Click on Manual and copy the Http url Paste it into the Request URL field in Expense Tracker.



        Step 4: Redirect URIs
        Enter the Expense Tracker site URL in both Redirect URI fields (Approved and Settings). Click Save to complete setup.


        Click on Users, then click on Settings. The settings panel will open, and at the bottom, you will see Validate Users with Zoho Books. Click on Configure.


        Click on Sync Users from Zoho Books


        After clicking on Sync Users from Zoho Books, you will be redirected to this page. Enter your email and click Next to proceed.


        Mapping Expense Types

        Open the Expense Type panel.


        Select Zoho Books Expense Type.


        Map your master expense type in Expense Tracker to the Zoho Books expense type.


        Expense Synchronization
        After the final approver gives approval, the expense record is marked as Expense Synced in Zoho Books. The details are then visible in Zoho Books.



      How to Integrate Copilot with Expense 365

      Prerequisites
      • Enter https://copilotstudio.microsoft.com/ in any browser and create a Copilot agent.
      • To configure this integration, you need to have a Premium Power Automate license.

      Steps to Integrate Copilot with Expense Tracker 365

      1. Click on your Copilot agent.


      2. Click on Settings.



      3. Click on Knowledge and add your Knowledge Base.



      4. Open https://portal.azure.com/ and click on App registrations.



      5. Click on the New registration button at the top of the App registrations page.



      6. Enter a Name for your application, configure any additional settings as needed, and then click the Register button at the bottom.



      7. After registration, on the application overview page, copy the Application (client) ID — you will need this for authentication.



      8. Click on the Add a certificate or secret link under Client credentials.





      9. Add a description and choose an expiration period, then click Add. Once created, copy the Value of the client secret immediately (it will only be shown once).





      10. Click on Add a Redirect URI in the application overview.





        Select Single-page application (SPA).



      11. Enter the URL https://token.botframework.com/.auth/web/redirect and click the Configure button to save the settings.



      12. In the left-hand navigation pane, click on API permissions, then click Add a permission.



      13. Choose the appropriate API (e.g., SharePoint / Microsoft Graph).



      14. Select the required permission type (e.g., Application permissions).



      15. Search and add all necessary permissions (e.g., Sites.Read.All) and click Add permissions.



      16. After adding all required permissions, click Grant admin consent (requires admin privileges).



      17. Go back to Copilot Studio, navigate to Settings, select Security, and click on Authentication.



        Click on Authenticate manually.



        In the Service Provider dropdown, select Microsoft Entra ID V2 with client secrets.



      18. Enter the following details:
        • Client ID: The Application (client) ID copied from Azure.
        • Client Secret: The Client Secret Value created in Azure.



      19. In the Scopes field, enter: profile openid Files.Read.All Sites.Read.All.



        Click Save, then click Publish to make authentication settings live.
      20. In Security, select Web channel security.



        Under web channel security, copy the Secret 1 key.



      21. Go to Power Automate, click on My Flows, import the Copilot server solution, and click Edit.





      22. Disable the New Designer toggle and paste the copied Secret 1 value into Headers (Authorization: Bearer [Secret]).



      23. Click Save, then copy the HTTP GET URL under the "When an HTTP request is received" trigger.



      24. Return to the Expense Tracker 365 application settings, navigate to Integration > AI API Integration, enable the Copilot integration toggle, and paste the copied URL into the Copilot URL field.





      AI API Integration


        Azure OpenAI introduces an advanced feature that automatically processes receipts and bills added as attachments. It extracts relevant details, organizes the data, and fills in the required fields, making expense management seamless and efficient. Users can record, categorize, manage, and analyze business expenses with greater accuracy and speed.

        To enable this functionality, navigate to General Settings and open the AI API Integration panel.


        The panel provides configuration options for Azure OpenAI:

        • Enable Azure OpenAI — Toggle to activate or deactivate AI integration.

        • Azure AI Endpoint — REST API or service endpoint hosted on Microsoft Azure.
        • Azure AI API Key — Authentication token to securely connect with Azure AI services.
        • Azure AI Deployment Name — Identifier of the deployed model within your Azure resource.

          • To learn how to generate and retrieve your Endpoint, API Key, and Deployment Name refer this Guide: Configure Azure AI.

        Once configured, click Save to apply the integration settings.


        Expense attachments can then be added directly from the Home Page by selecting Add, or by using the Upload option.


        To enable uploads, toggle on Import Cards in Settings and select Upload from the Home page.


        Select Expense Attachment and upload the required receipts or supporting documents.


        The system will automatically extract the relevant details from the uploaded receipts and populate the necessary fields in the Expense Tracker application.
