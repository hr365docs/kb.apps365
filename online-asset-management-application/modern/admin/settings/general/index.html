    Email Template

    You can modify the email notification format for both assigned and returned assets by following these steps:

    1. Navigate to the Configuration section in the side panel.
    2. A Custom Email page will open, where you can personalize the email content.
    3. To customize the email body, click the Action button and make the changes.


    Clicking the Action button will open a panel where you can edit the email's body, subject, and other details. Once you have made your changes, click Save to apply them.


    Email Notifications

    Clients could set up email notifications for assigned or on return assets. You can also setup reminders through email notifications for asset expiry and return date of the assets.


    By default, the setting “To users on assigning assets” is enabled, and email notifications are sent. Enter an email ID and click the Submit button—both the admin and the specified email address will receive notifications when an asset is assigned to a user.

    Similarly, the “To users on returning assets” setting is enabled by default. When an asset is returned, the admin and the entered email ID will receive an email notification.

    The “To groups on assigning assets” setting is also enabled by default. Enter the email ID and click Submit to send notifications to both the admin and the specified email addresses when assets are assigned to a group.

    Likewise, the “To groups on returning assets” setting is enabled by default. Email notifications will be sent to the admin and the provided email IDs upon asset return from the group.


    Email Reminders or Automatic Notifications


    The “Enable Automatic Reminders” feature is enabled by default. The admin receives email alerts regarding asset expiry and asset returns based on the number of days configured in the settings.

    Recipients listed in the “Choose Email Address” field will also receive reminders. You may enter up to 4 email addresses, separated by semicolons (“;”) with no spaces.

    When a value is entered under “Set Return Date Reminder” and submitted, the system sends an email notification the specified number of days before the asset's return date.

    Similarly, the “Set Expiry / End Date Reminder 1” option sends a notification the given number of days prior to the asset’s expiry date.

    The “Set Expiry / End Date Reminder 2” feature functions in the same manner as Reminder 1, allowing for a secondary reminder.

    The person who installs the application is assigned the admin role by default and will automatically receive all system-generated email notifications and reminders.

    When creating an asset, it is mandatory to assign it to a user during the creation process.


    Once a user receives an email notification for asset approval, they are expected to complete the approval process. If the user fails to approve the asset within the designated time, an Acknowledgement Date Reminder 1 will be triggered.

    This reminder is automatically sent after the number of days specified in the settings, serving as a follow-up notification to prompt the user to take the necessary approval action.


    If the user still does not approve the asset after receiving the first reminder, a Second Reminder will be sent. This acts as an additional prompt, urging the user to complete the approval process within the specified timeframe.


    The third option is an automatic approval. If the user does not take action within the specified number of days, the system will automatically process the approval based on the selected timeframe.
